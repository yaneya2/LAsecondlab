## Описание проекта

В проекте реализован однослойный перцептрон для задачи бинарной классификации. Код из ноутбука был разделён на несколько файлов: отдельно вынесены класс перцептрона, создание датасета, основной запуск обучения с визуализациями и эксперименты. Реализация соответствует требованиям ТЗ: генерация датасета, стандартизация признаков, разбиение на обучающую и тестовую выборки, обучение перцептрона с мини-батчами, построение графиков loss, подсчёт accuracy и проведение экспериментов с параметрами обучения.

```text
lab2_perceptron_project/
```

## Описание файлов

### `perceptron.py`

Файл содержит класс `Perceptron`.

В классе реализованы:

```text
__init__
sigmoid
forward
compute_loss
fit
predict
```

Перцептрон использует сигмоидную функцию активации. В качестве функции потерь используется бинарная кросс-энтропия. Обучение происходит с помощью градиентного спуска по мини-батчам. Во время обучения сохраняются значения ошибки на обучающей и валидационной выборках.

### `dataset.py`

Файл отвечает за создание и подготовку данных.

В нём создаётся датасет для бинарной классификации с параметрами:

```text
n_samples = 500
n_features = 2
n_redundant = 0
n_informative = 2
random_state = 42
n_clusters_per_class = 1
```

Также выполняется стандартизация признаков и разбиение данных на обучающую и тестовую выборки в соотношении 70% / 30% со стратификацией.

### `main.py`

Основной файл запуска.

В нём выполняется:

```text
создание датасета
создание модели
обучение модели
построение графика loss
подсчёт accuracy на train и test
визуализация разделяющей границы
```

Базовые параметры обучения:

```text
learning_rate = 0.1
epochs = 100
batch_size = 32
```

### `experiments.py`

Файл с экспериментами.

В нём проверяется влияние:

```text
скорости обучения
размера батча
инициализации весов
```

Для каждого эксперимента строится график loss и сохраняются итоговые значения accuracy, train loss и validation loss.

## Результаты базового обучения

При обучении модели с параметрами:

```text
learning_rate = 0.1
epochs = 100
batch_size = 32
```

значение функции потерь постепенно уменьшается. В начале обучения loss меняется быстрее, потому что веса ещё далеки от оптимальных значений. Ближе к последним эпохам изменение loss становится меньше, так как модель приближается к минимуму функции потерь.

По результатам обучения модель достаточно хорошо разделяет классы. Так как используется однослойный перцептрон, разделяющая граница является прямой линией. Это подходит для данных, которые можно примерно разделить линейно.

## Эксперимент 1. Влияние скорости обучения

Проверялись значения:

```text
0.001
0.01
0.5
1.0
2.0
5.0
```

Итоговые результаты из ноутбука:

| learning rate | accuracy | final Train Loss | final Val Loss |
|---:|---:|---:|---:|
| 0.001 | 0.893333 | 0.515919 | 0.521047 |
| 0.01 | 0.880000 | 0.348744 | 0.337745 |
| 0.5 | 0.886667 | 0.330826 | 0.300468 |
| 1.0 | 0.886667 | 0.330882 | 0.300182 |
| 2.0 | 0.886667 | 0.331112 | 0.299463 |
| 5.0 | 0.886667 | 0.334968 | 0.298144 |

### Вывод

При маленьком значении learning rate, например `0.001`, обучение идёт медленно. Это видно по высокому итоговому значению loss: модель не успевает достаточно сильно изменить веса за 100 эпох. Маленький шаг обновления делает обучение стабильным, но слишком долгим.

При `learning_rate = 0.01` ошибка уменьшается быстрее, чем при `0.001`, но всё равно итоговый loss выше, чем при больших значениях скорости обучения.

При больших значениях learning rate, например `0.5`, `1.0`, `2.0`, модель быстрее достигает меньшего значения функции потерь. Однако при слишком большом шаге обучение может стать менее стабильным: модель может перескакивать через минимум, из-за чего около минимума возникают колебания.

В данном эксперименте значения `0.5`, `1.0`, `2.0` и `5.0` дали похожую accuracy, но по loss видно, что слишком маленький learning rate хуже подходит для такого количества эпох. Оптимальным вариантом здесь можно считать learning rate из диапазона `0.5–1.0`, так как он даёт быструю сходимость и стабильное качество.

## Эксперимент 2. Влияние размера батча

Проверялись значения:

```text
1
16
64
256
```

Итоговые результаты из ноутбука:

| batch size | accuracy | final Train Loss | final Val Loss |
|---:|---:|---:|---:|
| 1 | 0.886667 | 0.331549 | 0.299068 |
| 16 | 0.886667 | 0.330802 | 0.300697 |
| 64 | 0.886667 | 0.331054 | 0.303044 |
| 256 | 0.886667 | 0.336588 | 0.318138 |

### Вывод

При `batch_size = 1` используется стохастический градиентный спуск. Веса обновляются после каждого объекта, поэтому график loss получается более зашумлённым. Такое обучение может быстрее реагировать на отдельные примеры, но направление обновления получается менее стабильным.

При `batch_size = 16` и `batch_size = 64` обучение становится более устойчивым. Градиент считается уже по нескольким объектам, поэтому обновления весов меньше зависят от одного конкретного примера. Значения accuracy почти не отличаются от `batch_size = 1`, но график loss становится более ровным.

При `batch_size = 256` график loss становится более гладким, потому что обновление считается по большой части выборки. Но из-за этого обновлений за одну эпоху меньше, поэтому модель может сходиться медленнее. Это видно по большему итоговому validation loss.

По итогам эксперимента accuracy оказалась одинаковой для всех размеров батча, но по функции потерь лучше выглядят небольшие и средние батчи. Самый большой batch даёт более гладкое, но более медленное обучение.

## Эксперимент 3. Влияние инициализации весов

Проверялись варианты:

```text
zero
small
large
```

Итоговые результаты из ноутбука:

| init type | accuracy | final Train Loss | final Val Loss |
|---|---:|---:|---:|
| zero | 0.886667 | 0.330802 | 0.301137 |
| small | 0.886667 | 0.330796 | 0.300973 |
| large | 0.866667 | 0.619155 | 0.403575 |

### Вывод

Нулевая инициализация для одного перцептрона допустима, потому что в модели нет нескольких одинаковых нейронов, между которыми нужно нарушать симметрию. Поэтому вариант `zero` смог обучиться и дал accuracy `0.886667`.

Маленькие случайные веса показали немного лучший результат по loss. Такая инициализация удобна, потому что модель начинает обучение рядом с нулём, но при этом веса уже не полностью одинаковые. В данном эксперименте `small` дал лучший validation loss среди вариантов инициализации.

Большая инициализация с крупными значениями весов сработала хуже. При больших весах значение `z = wT x + b` может становиться слишком большим по модулю. Тогда сигмоида переходит в область насыщения, где её производная близка к нулю. Из-за этого градиенты становятся маленькими, веса обновляются хуже, и обучение замедляется. Это видно по результатам: у `large` выше train loss и validation loss, а accuracy ниже.

Таким образом, для данной задачи лучше всего подходит инициализация маленькими случайными значениями.

## Общий вывод

В ходе работы был реализован однослойный перцептрон для бинарной классификации. Модель обучалась без использования готовых библиотек глубокого обучения. Были реализованы основные части алгоритма: прямой проход, сигмоида, бинарная кросс-энтропия, обновление весов и предсказание класса.

На базовом наборе данных перцептрон показал хорошее качество, потому что данные имеют два признака и могут быть достаточно хорошо разделены линейной границей. Разделяющая граница перцептрона является прямой, поэтому такая модель подходит для линейно разделимых или почти линейно разделимых данных.

Эксперименты показали, что скорость обучения сильно влияет на сходимость. Слишком маленький learning rate приводит к медленному обучению. Большие значения learning rate ускоряют обучение, но могут вызывать колебания около минимума.

Размер батча влияет на характер графика loss. Маленький batch даёт более шумное обучение, потому что веса обновляются часто и по малому числу объектов. Большой batch делает график более гладким, но может требовать больше эпох для достижения хорошего результата.

Инициализация весов также влияет на качество обучения. Нулевая инициализация для одного перцептрона допустима, маленькие случайные веса дают стабильное обучение, а слишком большие веса могут привести к насыщению сигмоиды и ухудшению качества.

Лучший вариант для данной задачи — использовать маленькую случайную инициализацию весов, умеренный размер батча и не слишком маленькую скорость обучения.
